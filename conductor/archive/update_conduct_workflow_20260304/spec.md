# Specification: Update Conduct Workflow (Git Sync)

## Overview
Update the `conductor/workflow.md` to include mandatory Git synchronization steps. This will ensure that the environment is always up-to-date before starting work and that changes are pushed immediately after completion, preventing merge conflicts when working across multiple machines.

## Functional Requirements
1. **Mandatory Phase Start Sync:** Add a step at the beginning of each Phase in the workflow to perform a `git pull --rebase`.
2. **Mandatory Phase Completion Sync:** Add a step at the end of the "Phase Completion Verification and Checkpointing Protocol" to perform a `git push`.
3. **Conflict Handling:** 
   - The workflow must specify that the agent should first attempt to handle trivial conflicts during `git pull --rebase`.
   - If conflicts are complex or involve significant code changes, the agent must halt and ask the user for guidance.
4. **Protocol Integration:** Seamlessly integrate these steps into the existing "Standard Task Workflow" and "Phase Completion Verification and Checkpointing Protocol" sections.

## Acceptance Criteria
- `conductor/workflow.md` is updated with the new Git sync requirements.
- The instructions for "Phase Completion" clearly state the requirement to push after the checkpoint commit.
- Future implementation plans generated by Conductor will include these sync tasks.

## Out of Scope
- Automatic resolution of complex code conflicts.
- Synchronization at every individual task level (unless specified otherwise).
